熱門遠端儲存庫(Github VS Bitbucket VS Gitlab)

  • GitHub：擁有 GitHub Pages 功能，可擁有私人數據庫，免費方案是 3 人以下
  • Bitbucket：可擁有私人數據庫，免費方案是五人以下團隊
  • GitLab：自架 Git 伺服器，有提供 web 視覺化管理介面，常用於企業內部開發

遠端儲存庫(Repository)操作

  • 註冊遠端儲存庫：git remote add origin 遠端儲存庫網址

  • 更新資料到遠端 master 分支：git push -u origin master

  • -u 是指他預設會推到哪個遠端數據庫服務

  • origin 可以改它的遠端數據庫名稱，例如 git push -u github master

origin 是預設遠端數據庫名稱，所以你可以做更改
開了一個遠端數據庫之後，可以在裡面新增另一個數據庫，要記得名稱不要重複即可

開多個遠端數據庫是為了可能有個數據庫是主要正式主機(已上線)，另一個為測試用數據庫、或是開發用

本地端開發 → push 到測試用數據庫 (git push git2 master) → code review 以及測試後沒問題才會部屬到正式上線主機

剛開始的 branch 是預設叫做 master

Git 版本細節

  • branch ：分支，預設分支叫做 master
  • HEAD：指標 (現在位置)
  • origin：預設遠端儲存庫名稱
  • 回頭觀看版本內容：git checkout 編號
  • 返回最新的版本：git checkout master(分支名稱)

可以用 git log 查看 commit 歷史紀錄
版本編號只要打前幾碼 + tab 就可以補齊版本編號 (git checkout 版本編號)
或是直接在 source tree 上面看

也可以在本地端的資料庫點進半透明的 .git 資料庫的 refs 去看

還原技巧

  • 顯示此次清除的檔案：git clean -n
  • 強制清除檔案：git clean -f

檔案已加入追蹤，清空工作目錄

  • 還原工作目錄上已更改的檔案 ：git checkout –

檔案加入到索引，退到工作目錄

  • 加入索引的檔案還原到工作目錄：git reset HEAD

版本還原

  • 還原前兩個版本：git reset HEAD^^
  • 還原前兩個版本，所有更新檔案都放棄：git reset HEAD^^ –hard
  • 觀看詳細歷史紀錄：git reflog
  • 還原到特定 commit：git reset commit編號 –hard
  • git reset 參數介紹 (https://gitbook.tw/chapters/using-git/reset-commit.html)
