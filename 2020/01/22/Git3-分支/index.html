為什麼要用分支？

  • 多人協作時，不可能都在 master，會依照各自的分工在不同的分支上開發
  • 可以讓 master 都是正式版資料，可以開分支來做測試或開發，藉此不影響正式主機分支

開分支流程

新增分支：git branch 分支名稱
查看分支：git branch
切換分支：git checkout 分支名稱
刪除分支：git branch -d 分支名稱、-D 是強制刪除
還原上個版本：git reset HEAD^ (以 HEAD 當基準上一步^，上上一步 ^^)

開發情境：我有一個正式主機 master 分支，有一個開發分支叫做 develop

合併分支 && 快轉機制

合併分支：git merge 分支名稱
取消快轉：git merge 分支名稱 –no-ff
觀看線圖：git log -oneline -graph
還原合併前狀態：git reset -hard ORIG_HEAD

讓 master 合併 develop，必須先切換到 master branch(git merge develop)

快轉：使用 merge 之後，master 的進度直接跳到 develop 的 commit

取消快轉：使用 git merge develop –no-ff，這樣狀況下 develop 會保留自己的 commit，master 會自己再創造出新的 commit，但內容是 develop 的進度
以下這張圖的本地端就是取消快轉的效果，會多出一個 commit
如果是直接 git merge master develop 的話，master 就會直接併到 develop 的地方，這樣就沒辦法留一個紀錄是 develop 那時開發了甚麼

只要要合併的兩個 branch 處在同一條線上，commit 就會出現快轉，但如果兩個 branch 分別處在不同的時空下(?)，用源頭去想會比較好理解，就不會產生快轉，所以這時只要普通的 merge 就好

假如我們現在在 dev 上，那照 dev 的線往源頭找，如果沒有碰到 master，那此次 merge 就不會有快轉的狀況出現
